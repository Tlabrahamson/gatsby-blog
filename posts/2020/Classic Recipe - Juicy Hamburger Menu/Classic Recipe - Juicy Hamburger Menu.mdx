Ahh yes, the hamburger menu. A classic and essential component of many a recipe. The hamburger menu is versatile and can be crafted a number of ways.

<br/>


**Ingredients:**

*HTML*
*CSS*
*A dash of JavaScript to toggle the flavor*

<br/>

Let's get started shall we? First we need to create a project folder. Call it whatever you'd like! I'm going to go with "Juicy Burger Menu". Now let's open the project in our favorite text editor and begin cooking. 

<br/>


**Let's add three files to our project folder:**

*index.html*
*style.css*
*burger.js*

<br/>

Let's start creating our index.html with a little ! + tab to get a basic HTML document going. We'll go ahead and link our stylesheet and script. We'll also chang the title of our page to Juicy Burger Menu.


```jsx
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <meta http-equiv="X-UA-Compatible" content="ie=edge" />
    <link rel="stylesheet" href="./style.css" />
    <title>Juicy Burger Menu</title>
  </head>
  <body>
    <script src="./hamburger.js"></script>
  </body>
</html>

```


Lovely. Now we can work on our header and navigation. We will wrap everything in a header. Inside of the header we are going to have a div with an h1 inside of it that says Logo. Under the div, we want a nav element. Since we are building our hamburger from scratch, we are going to add a div with a class of "hamburger" in that nav. Inside of the "hamburger" div, we are going to add three more divs with the classes "line-1", "line-2", and "line-3". Lastly, under the "hamburger" div, we are going to add an unordered list with the class "nav-list" with four list items each with an anchor element with the class "nav-link". We can make the href on the anchors "#" and have the text Blog, Shop, About, and Contact. Something like that.


```jsx
<header>
    <div class="logo">
        <h1>Logo</h1>
    </div>
    <nav>
        <div class="hamburger">
            <div class="line-1"></div>
            <div class="line-2"></div>
            <div class="line-3"></div>
        </div>
        <ul class="nav-list">
            <li><a class="nav-link" href="#">Blog</a></li>
            <li><a class="nav-link" href="#">Shop</a></li>
            <li><a class="nav-link" href="#">About</a></li>
            <li><a class="nav-link" href="#">Contact</a></li>
        </ul>
    </nav>
</header>

```

That'll do it for our HTML. Let's set it aside and work on our CSS.

<br/>

Let's open up our style.css and do a basic reset on the document by using the universal selector to set the margin and padding to 0 and the box-sizing to border-box.

```jsx
* {
  margin: 0;
  padding: 0;
  box-sizing: border-box;
}

```

Now we are going to focus on the layout of the page. At the moment, everything is bunched together. We can begin to make the header look nice by displaying it as flex, justify-content space between, align items center, and give it 1rem padding on the top and bottom and 4rem left and right.

```jsx

header {
  display: flex;
  justify-content: space-between;
  align-items: center;
  padding: 1rem 4rem;
}
```

Let's get the nav-list and remove the list styles, display it as flex, and justify content space-evenly. We can also get the nav-links and remove the text-decoration, give it padding of 1rem, increase the font-size, and set the color to inherit.

```jsx
.nav-list {
  list-style: none;
  display: flex;
  justify-content: space-evenly;
}

.nav-link {
  text-decoration: none;
  justify-content: space-between;
  padding: 1rem;
  font-size: 1.25rem;
  color: inherit;
}

```

Since our focus is the hamburger, I'm just going to throw down some basic styling on the elements below the header.

```jsx
main {
  margin: 2rem auto;
  padding: 0rem 4rem;
  width: 800px;
}

article {
  margin: 4rem 0;
}

article h2 {
  font-size: 2.5rem;
}

article p {
  font-size: 1.25rem;
}

```

Mmmmmhmmm. That's really coming along. Problem is, if the screen is smaller, everything will get jumbled up or come off of the screen completely. That means that it is media query time. We are going to have just one media query that checks if the max width is less than 800px.

<br/>

Let's loosen up the padding on the header and make our hamburger visible.

```jsx
@media screen and (max-width: 800px) {
  header {
    padding: 1rem 2rem;
  }

  .hamburger {
    display: block;
    cursor: pointer;
  }

  .line-1,
  .line-2,
  .line-3 {
    width: 24px;
    height: 4px;
    background: #000;
    margin: 2px 0;
  }
}

```

We gave the lines some dimension and added a background color as well as some margin. Now we are really jumbled! We can make the nav-list disappear by displaying it as none, but that won't allow us to do any transition animation later. SO, we can just push it off the screen by making it's position absolute. We will also go ahead and make the main text container responsive.

```jsx
.nav-list {
    position: absolute;
    top: -50%;
    left: 50%;
}

main {
    width: 100%;
    padding: 0rem 2rem;
}

```

Now we are cooking! Nothing will work without functionality though. Let's crack open our hamburger.js and add some! We want to get a hold on our hamburger class that wraps the three lines. We also want our nav-list.

```jsx
const hamburger = document.querySelector(".hamburger");
const navList = document.querySelector(".nav-list");

```

Now that we have both of our classes stored in the constants hamburger and navList, we want to trigger an event each time the hamburger is clicked. Let's add an event listener to our hamburger constant and pass in the click event and a function that we will call toggleMenu.

```jsx
hamburger.addEventListener('click', toggleMenu);

```

For the toggleMenu function, we want to toggle a class on our navList called 'active' each time the hamburger is clicked. It is short and sweet!

```jsx
const toggleMenu = () => {
  navList.classList.toggle("active");
};

```

Since I used an arrow function, our function must be above the event listener. The file should look something like:

```jsx
const hamburger = document.querySelector(".hamburger");
const navList = document.querySelector(".nav-list");

const toggleMenu = () => {
  navList.classList.toggle("active");
};

hamburger.addEventListener("click", toggleMenu);
```

That's it for our JavaScript! Or is it? For now, let's get back in our style.css and create the active class.

```jsx
.active {
    top: 0;
    height: 100vh;
}

```

The active class will set the top to 0 as opposed to the -50% that we have to move it off screen. We give it a height of 100vh so it will take up the entire height of the screen when it is active. So now if we click our hamburger...

<br/>

I guess we should have cooked it longer. We can fix this! Let's start by updating our nav-list class in our media query.

```jsx
.nav-list {
    position: absolute;
    top: -50%;
    left: 50%;
    transform: translate(-50%);
    height: 0;
    background: #fff;
    z-index: 1;
    flex-direction: column;
    align-items: center;
    width: 100%;
}

```

**hamburger.js**

```jsx
const hamburger = document.querySelector(".hamburger");
const navList = document.querySelector(".nav-list");
const body = document.querySelector("body");

const toggleMenu = () => {
  navList.classList.toggle("active");
  body.classList.toggle("body-active");
};

hamburger.addEventListener("click", toggleMenu);
```
